## Instances that Are Removed From a Hosting Environment for Misfeature

Every instance that is removed from your instance group for a legitimate reason must be removed from your instance group for a reason that you do not want replicated when running in your hosted environment. This can usually occur through the following situations:

* Your instance&#x27;s capacity has been exhausted
* Your host server has not been updated for 10 years
* Your version of Istio is no longer supported
* Your database configuration does not use an Istio database
* Your instance has been marked for removal.
This document describes the important configurations that you must have when running in a hosting environment.
1. **Be sure** that all instances have been added to your DFSG.

`aws ec2 describe-instances --filters &quot;Instance-Id eq &#x27;(your Istio Instance Id)&#x27; --filter-attribute Name &#x27;Dfsg.Name&#x27;&quot; --region-name &#x27;us-east-1&#x27;`

2. **Don&#x27;t have more than one instance in your DFSG**.
